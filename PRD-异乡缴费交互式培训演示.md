# PRD：异乡缴费交互式 HTML 培训演示

> **创建日期**：2026-03-18
> **负责人**：Neil Ding
> **状态**：Draft — 待审阅后交付 AI 开发

---

## Problem Statement

异乡缴费团队每月需要对异乡好居的租赁顾问进行业务培训（15 分钟线下宣讲 + 投屏展示）。当前使用传统 PPT 作为培训载体，存在以下痛点：

1. **视觉表现力不足**：PPT 的静态排版难以生动呈现比价优势、流程逻辑等核心信息，顾问容易走神。
2. **信息传递效率低**：关键对比数据（比价平台 vs 单一平台）在 PPT 中只是一张静态表格，缺乏逐行对比的节奏感，讲师需要反复口头强调。
3. **品牌一致性弱**：PPT 风格与 uhomespay.com 官网的现代化视觉语言脱节，削弱了品牌专业感。
4. **维护成本高**：PPT 文件在多人流转中容易出现版本混乱，格式错位。

---

## Solution

构建一个**单文件交互式 HTML 培训演示页面**，采用 reveal.js 幻灯片框架，内嵌品牌主题样式和步进动画，替代传统 PPT。

**核心体验**：
- 讲师（Neil）在电脑上双击打开 `.html` 文件，全屏投影，用键盘/点击翻页讲解
- 每个章节配有过渡动画和步进展示（fragment），让信息按节奏逐步呈现
- 比价对比表采用交互式逐行浮出设计，先展示「单一平台」劣势，再逐行揭示「异乡缴费」优势，制造对比冲击
- 视觉风格对齐 uhomespay.com 官网（渐变、现代卡片、脉冲动画），强化品牌认知

**技术约束**：
- 单个 `.html` 文件，所有 CSS/JS 内嵌或通过 CDN 引用
- 图片资源通过 CDN 外链加载（合作方 logo 等参考 uhomespay.com 线上素材）
- 无需服务器部署，无需构建工具，双击即用
- 每月由 Neil 审阅后交给 AI 更新内容

## User Stories

1. As a **培训讲师（Neil）**，I want to 双击打开一个 HTML 文件就能开始投屏演示，so that 不依赖 PowerPoint 软件，任何电脑都能用。

2. As a **培训讲师**，I want to 用键盘方向键或点击来控制幻灯片翻页和步进，so that 演讲节奏完全由我掌控。

3. As a **培训讲师**，I want to 每个章节的要点能逐条步进展示（fragment），so that 观众跟着我的讲解节奏逐步吸收信息，而不是一次看到所有内容。

4. As a **培训讲师**，I want to 比价对比表能先展示「单一平台」的劣势，按键后「异乡缴费」列逐行浮出对比，so that 制造视觉冲击力，让顾问直观感受比价优势。

5. As a **培训讲师**，I want to 页面视觉风格与 uhomespay.com 官网一致，so that 培训时传递统一的品牌形象，让顾问感受到产品的专业度。

6. As a **租赁顾问（受训者）**，I want to 看到流畅的动画和清晰的信息层次，so that 15 分钟培训内容不枯燥，核心信息能记住。

7. As a **租赁顾问**，I want to 在「平台介绍」章节看到四大优势（官方通道/人民币支付/不占外汇/快速到账）逐一呈现，so that 我能逐个理解每个优势点。

8. As a **租赁顾问**，I want to 在「缴费类型」章节清晰看到学费、房租、保险三种缴费场景，so that 我知道可以推荐哪些缴费服务。

9. As a **租赁顾问**，I want to 在「合作机构」章节看到 Flywire、Convera 等持牌机构的 logo 和合规声明，so that 我确信推荐的渠道是合法合规的，可以放心引导租客。

10. As a **租赁顾问**，I want to 在「成单流程」章节看到流程图逐步展开（申请→合同→支付→录单），so that 我理解支付定金是成单的关键节点。

11. As a **租赁顾问**，I want to 看到推荐异乡缴费的前期价值（加速成单）和后期价值（防逾期保合同），so that 我理解推荐的核心动机不只是佣金。

12. As a **租赁顾问**，I want to 在「推荐理由」章节看到三大理由以卡片形式逐一呈现，so that 信息结构清晰、易记。

13. As a **租赁顾问**，I want to 在「奖励机制」章节看到具体的佣金比例和计算示例（如 10 万学费 → 30 元奖金），so that 我清楚推荐后的实际收益。

14. As a **租赁顾问**，I want to 在「内部对接」章节看到两种场景的流程对比（自动商机分配 vs 手动喊单），so that 我知道什么情况走哪条路径。

15. As a **培训讲师**，I want to 最后一页有清晰的「答疑」引导和联系方式，so that 自然过渡到互动环节。

16. As a **培训讲师**，I want to 每月更新内容时只需修改 HTML 中的文本和数字，结构不变，so that 维护成本低，可以交给 AI 快速完成。

17. As a **培训讲师**，I want to 页面在 1920×1080 分辨率下全屏展示时排版完美，so that 投屏效果专业、清晰。

18. As a **培训讲师**，I want to 看到幻灯片底部有页码/进度指示，so that 我知道培训进度，控制时间。

## Implementation Decisions

### 技术选型

| 决策 | 选择 | 理由 |
|------|------|------|
| 幻灯片框架 | **reveal.js**（CDN 引入） | 成熟的 HTML 演示框架，支持 fragment 步进、过渡动画、全屏模式、键盘控制 |
| 文件结构 | **单个 .html 文件** | 双击即用，零依赖，便于分发和版本管理 |
| 样式方案 | **内嵌 `<style>` 自定义主题** | 对齐 uhomespay.com 品牌风格，覆盖 reveal.js 默认主题 |
| 图片资源 | **CDN 外链**（合作方 logo、装饰图） | 保持文件轻量，利用 uhomespay.com 等线上已有素材 |
| 动画方案 | **reveal.js fragment + CSS animation** | fragment 控制步进节奏，CSS animation 实现入场/强调效果 |

### 品牌风格规范

- **主色调**：参考 uhomespay.com 品牌色系（红/橙暖色 + 深色背景 or 浅色背景）
- **字体**：系统中文字体栈（`"PingFang SC", "Microsoft YaHei", sans-serif`）
- **卡片风格**：圆角 + 轻阴影 + hover 微交互（投屏场景下主要服务于视觉层次）
- **动画原则**：克制，服务于信息节奏，不炫技；每个 fragment 步进时间 0.3-0.5s

### 幻灯片结构（10 章节）

#### Slide 1 — 封面
- 异乡缴费 logo + Tagline「留学缴费 比一比 更省钱」
- 品牌动画入场（logo 缩放 + 文字淡入）
- 底部：培训人信息 + 日期

#### Slide 2 — 平台介绍
- 一段话介绍异乡缴费核心定位
- 四大优势以卡片/图标形式 **fragment 逐一步进**：
  1. 官方付款通道
  2. 人民币快捷支付
  3. 不占外汇额度
  4. 快速到账服务（最快 3 小时）

#### Slide 3 — 缴费类型
- 三大缴费场景：**学费 / 房租 / 保险**
- 每个场景配图标 + 简要说明
- 底部展示支持的支付方式 logo 墙（支付宝、微信、VISA、MasterCard 等）

#### Slide 4 — 为什么是比价平台（核心交互页）
- 标题：「比一比 更省钱 — 为什么是比价平台而不是支付平台？」
- **交互式对比表**：
  - 初始状态：只显示「单一支付平台」列（6 行：选择权/价格透明度/手续费/合规性/风险/服务中立性），内容偏负面（红色/灰色调）
  - 按键/点击后：「异乡缴费比价平台」列 **逐行从右侧滑入**，每行绿色高亮，形成鲜明对比
  - 最后一步：底部出现总结语「中立第三方顾问，为留学生找到最优方案」

#### Slide 5 — 合作机构与合规
- 合作机构 logo 墙（Flywire、Convera、TransferMate、支付宝、微信支付等）
- logo 以 **渐次淡入** 方式呈现
- 合规声明文字 fragment 步进（强调「持牌合规金融机构」）

#### Slide 6 — 顾问成单流程
- **前期流程**：用流程图展示两条成单路径，**逐步展开**
  - 流程一：申请 → 出合同 → 支付定金 → 接受合同 → 录单
  - 流程二：申请 → 支付定金 → 出合同 → 签署 → 上传资料 → 录单
- **高亮标注**「支付定金」为关键节点
- **后期风险**（fragment 步进）：未及时提醒 → 客户逾期 → 公寓取消合同 → 客诉 + 扣奖金
- **推荐原因**（fragment）：效率提升 / 过程可控 / 成单保障

#### Slide 7 — 推荐三大理由
- 三栏卡片布局，**逐一步进**：
  1. 🕐 降低时间成本，促成单（签约→缴费→售后闭环）
  2. 💼 解决缴费问题，更专业（跨境缴费痛点 + 全程代办）
  3. 💰 促成其他缴费业务，提佣金（内部推荐奖励）

#### Slide 8 — 奖励机制
- 三个奖励模块 **fragment 步进**：
  - **学费佣金**：支付宝 15-30 元/单；飞汇/Convera 0.03%-0.05%
  - **保险佣金**：澳洲保险 1%；美国替换保险 50-100 元/单
  - **汇款阶梯奖励**：澳洲/香港汇款按月累计
- 计算示例动画：「10 万学费 × 0.03% = 30 元奖金」数字高亮滚入
- 总结语 fragment：「内部同事多多推荐奖励多多！」

#### Slide 9 — 内部对接模式
- **双路径流程图**（fragment 分两步展示）：
  - 场景 1（自动）：租赁订单 → 系统自动产生缴费商机 → 分配缴费顾问 → 拉群对接
  - 场景 2（手动）：系统未产生商机 → 在国家对接群内喊单 → 缴费顾问认领
- 近期优化提示：「即将上线 1 对 1 提醒功能」

#### Slide 10 — 答疑与结束
- 大标题：「问题答疑」
- 联系方式 / 内部对接群二维码（如有）
- 结束语 + 品牌 logo 动画收尾
- 备注：考核环节在北森系统完成（本页可加一句引导语提示）

### 内容更新策略

- 所有可变文本（佣金数字、流程描述、合作方列表）集中放在 HTML 文件顶部的 `<!-- CONFIG START -->` 区域，或以 JS 变量形式集中管理
- 每月更新时只需修改这些集中区域的值，无需改动结构和样式
- 更新后双击打开验证即可

## Testing Decisions

### 什么是好的测试

本项目是一个单文件 HTML 演示页面，不涉及后端逻辑或数据库。测试重点在于**视觉呈现和交互行为的验收**，而非自动化单元测试。

### 验收测试清单（人工）

| 测试项 | 验收标准 |
|--------|----------|
| **文件可用性** | 双击 `.html` 文件在 Chrome/Edge 中正常打开，无控制台报错 |
| **全屏投屏** | 1920×1080 分辨率全屏模式下，所有文字清晰可读，布局无溢出 |
| **键盘导航** | 方向键左右翻页、上下步进，ESC 退出全屏，均正常工作 |
| **Fragment 步进** | 每个章节的 fragment 按预期顺序逐条展示，动画流畅 |
| **比价对比表交互** | 初始只显示「单一平台」列；按键后「异乡缴费」列逐行浮出，颜色对比鲜明 |
| **品牌一致性** | 配色、字体、卡片风格与 uhomespay.com 视觉语言一致 |
| **CDN 图片加载** | 合作方 logo 全部正常加载，无 broken image |
| **培训时长** | 以正常讲解速度走完全部幻灯片，耗时 ≤ 15 分钟 |
| **内容准确性** | 佣金数字、流程描述、合作方名称与 PPT 原始内容一致 |

### 可选的自动化验证

如后续需要，可用 Playwright 编写简单的端到端测试：
- 打开 HTML → 验证 slide 数量 = 10
- 模拟键盘翻页 → 验证每页标题文本
- 验证比价表交互状态切换

当前阶段不需要自动化测试。

## Out of Scope

1. **考核/考试功能** — 考核环节在北森系统处理，不在本 HTML 中实现
2. **服务器部署** — 当前仅本地文件使用，不考虑部署到 Web 服务器
3. **离线图片内嵌** — 图片通过 CDN 加载，不做 Base64 内嵌（需联网环境）
4. **移动端适配** — 目标场景为电脑投屏，不需要手机/平板适配
5. **多语言** — 当前仅中文版本
6. **顾问自助学习模式** — 当前仅讲师投屏场景，不考虑顾问自行浏览的场景
7. **内容管理后台** — 内容更新直接编辑 HTML 文件，不建设 CMS
8. **GitHub 仓库托管** — 当前不考虑代码仓库管理

## Further Notes

- 浏览器：Chrome 或 Edge（最新版），需联网加载 CDN 资源
- 分辨率：1920×1080（投影仪标准分辨率）

- 主要内容来自 PPT `新人第二天内部培训.pptx`（10 页）
- 品牌素材参考 uhomespay.com 官网
- 合作方 logo 从线上公开渠道获取

- 如培训频率增加或受训人数增长，可考虑部署到内部站点，支持顾问自助学习
